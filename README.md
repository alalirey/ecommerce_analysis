# Анализ данных e-commerce платформы (Olist)

## Описание проекта

В рамках проекта был проведён комплексный анализ данных интернет-платформы электронной коммерции на основе датасета Olist.

Цель проекта — исследовать:
- динамику заказов
- логистические процессы
- финансовые показатели
- поведение клиентов

Проект демонстрирует навыки анализа данных с использованием Python и SQL.

---

## Используемые инструменты

- Python:
  - pandas
  - matplotlib
  - seaborn
- SQL (PostgreSQL)
- Jupyter Notebook

---

## Основные этапы анализа

### 1. Предобработка данных
- обработка пропусков
- выявление и обработка аномалий (временные несоответствия)
- создание новых признаков:
  - статусы заказов
  - временные интервалы (обработка, доставка)
  - флаг аномалий


### 2. Описательный анализ (Descriptive Analytics)

#### Анализ заказов
- распределение заказов по статусам
- выявление несоответствий между статусами и датами
- 
#### Временной анализ
- анализ динамики заказов по месяцам
- выявление сезонных пиков (ноябрь)
- анализ активности по дням недели и времени

#### Логистика
- анализ времени обработки заказов
- анализ времени доставки
- расчёт доли опозданий (~8%)
- анализ задержек доставки
- выявление асимметрии распределения

---

### 3. Финансовый анализ (SQL)
- анализ выручки платформы
- выявление периодов роста
- связь между количеством заказов и выручкой
- расчёт агрегированных метрик

---

### 4. Анализ клиентов (SQL)
- количество уникальных пользователей (~96 000)
- доля повторных покупок (~3.12%)
- среднее количество заказов на клиента (~1.03)
- распределение пользователей по количеству заказов

---

## Основные выводы

- Наблюдается рост количества заказов с течением времени
- Был выявлен выраженный пик заказов в ноябре, вероятно связанный с сезонными акциями
  
- Большинство заказов доставляется в течение **5–12 дней**, медиана — около **7 дней**
- Около **8% заказов** доставляются с опозданием
- Логистика в целом стабильна, но присутствуют редкие значительные задержки

- Платформа имеет более **96 000 пользователей**
- Более **96% клиентов** совершают только одну покупку
- Доля повторных покупок составляет всего **3.12%**
- Основная зона роста — **удержание клиентов**

---

## Бизнес-инсайты

- необходимо снижать задержки доставки
- важно развивать стратегии удержания пользователей
- стоит использовать сезонные пики спроса для увеличения продаж

## Как запустить проект

1. Скачать датасет Olist (https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)
2. Установить зависимости:
   pip install pandas matplotlib seaborn
3. Запустить ноутбук: ecommerce_analysis.ipynb
